# Implement Spec Skill

仕様ファイルを起点に、機能の実装・修正を行う。

## 手順

1. `docs/specs/features/$0` の仕様ファイルを読み、概要・受入基準を確認する
2. 関連する `docs/specs/architecture.md` と `docs/specs/data-model.md` を参照し、アーキテクチャとデータモデルの制約を把握する
3. 仕様内の「実装ファイル」セクションから対象ソースコードを調査する
4. 実装方針を3行で説明し、ユーザーの承認を得てから実装を開始する
5. 実装後、`npx tsc --noEmit` と `npm test` で検証する
6. 仕様の受入基準をチェックリストとして報告する（各項目の達成/未達成を明示）
7. 変更があれば conventional commit でコミットし、ブランチを push して PR を作成する

## ルール

- 仕様に記載がない機能追加・リファクタをしない
- 受入基準をすべて満たすまで完了としない
- 仕様と実装に乖離が見つかった場合、仕様の更新を先に提案する
- 2回修正して解決しない場合は原因分析を報告する
- main への直接 push は禁止
