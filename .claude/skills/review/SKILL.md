# Review Skill

現在のブランチのPRを自動レビューする。

## 手順

1. `gh pr list --head $(git branch --show-current)` で現在のブランチのオープンPRを特定する（ユーザーにどのPRか聞かない）
2. `gh pr diff` でPRの差分を取得する
3. 以下の観点でレビューする:
   - バグ・ロジックエラー
   - TypeScript型安全性
   - エッジケースの見落とし
   - UI/スタイリングの問題
   - セキュリティ上の懸念
4. 構造化されたレビューを出力する:
   - **概要**: 変更の要約（2-3行）
   - **問題点**: ファイルパスと行番号付きで指摘
   - **改善提案**: 必須ではないが推奨する変更
5. 修正を適用するか確認する

## ルール

- PRが見つからない場合はその旨を伝える
- マージはユーザーの明示的な指示がない限り実行しない
